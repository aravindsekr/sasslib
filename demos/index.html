Front End System Design

This page shows the basic FE design and how different components interact.
Also you can view standards and styleguide on how to use these elements in the project.

Why React ?

  • Stable one with good support for long time
  • Virtual DOM makes updates faster and deterministic
  • React is declarative and functional
  • Who else is not using it ? :-)

Why SASS ?

  • SASS can make CSS files more oraganized
  • Use of mixins, variables and functions
  • Can write modular and resusable styles

Webpack

  • Config to build FE Assets
  • Development and Release modes makes debugging easy
  • Minification of files
